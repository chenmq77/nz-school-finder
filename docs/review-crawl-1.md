# GPT 挑战 — 爬虫方案第 1 轮

**日期**: 2026-03-22
**评分**: 6/10
**结论**: CHANGES_REQUESTED

## 关键问题

1. **[critical] 字段≠能稳定结构化提取** — 很多信息在 PDF、手册、第三方页面里，不是简单 HTML
2. **[critical] NCEA 成绩不应以学校官网为主源** — 学校自报口径不一致，应走 NZQA/Education Counts
3. **[critical] 数据架构过于扁平** — 单表丢失来源URL、置信度、时效性，应改为三层架构
4. **[major] 字段优先级需重排** — 学区/课程/费用 > 校服/开放日/校历
5. **[major] 缺少字段：学生支持/wellbeing、特殊教育、宗教特色、入学流程**
6. **[major] 过早绑定 Scrapling** — 学校网站难点是 PDF 提取和语义归一化，不是 CSS 选择器
7. **[major] StealthyFetcher 不应默认使用** — 学校网站非高对抗目标，应遵守 robots.txt
8. **[major] 应做分层样本 POC（6-10所）** 而非只测 2 所顶级学校
9. **[major] 缺少字段级抽取规范** — 每个字段需定义首选源→备选源→归一化→回退
10. **[minor] 测试学校选择偏乐观** — 应加入维护较弱的学校网站

## SUMMARY
方向正确，但应改为"权威公开源优先、官网补充、人工校验兜底"的数据产品思路。
