# 新西兰学校数据分类与展示方案（V2）

> 核心需求：用户输入学校英文名称 → 系统从 CSV（2576所学校、49列）中匹配 → 按分类有条理地展示学校信息
> 设计视角：以**家长/学生选校**的真实决策流程为导向


## 一、分类总览（5大类）

站在家长/学生角度，选校的思维链是：

> **① 这是什么学校？→ ② 在哪？怎么联系？→ ③ 学生情况如何？→ ④ 学校质量/特色怎么样？→ ⑤ 属于哪个区域管辖？**

| 大类 | 中文名 | 列数 | 展示方式 | 家长关心程度 |
|------|--------|------|----------|-------------|
| A | 学校身份概览 | 7列 | 默认展开 | ⭐⭐⭐⭐⭐ |
| B | 位置与联系方式 | 10列 | 默认展开 | ⭐⭐⭐⭐⭐ |
| C | 学生规模与构成 | 9列 | 默认展开 | ⭐⭐⭐⭐ |
| D | 学校质量与特色 | 11列 | 点击展开 | ⭐⭐⭐ |
| E | 行政与区域归属 | 12列 | 点击展开 | ⭐⭐ |

**共计 49 列，全部覆盖。**


## 二、每大类的详细字段

### A. 学校身份概览（7列）— 一眼判断"这学校适不适合我"

家长最先想知道：叫什么名字？公立还是私立？男校女校？收不收寄宿？官网在哪？

| 序号 | 中文名 | CSV原始列名 | 展示建议 |
|------|--------|-------------|----------|
| A1 | 学校名称 | School Name | 页面标题，加粗大字 |
| A2 | 学校编号 | School Number | 小字灰色标注在名称旁 |
| A3 | 学校类型 | School Type | 如"中学(9-15年级)"、"小学"、"综合学校" |
| A4 | 管理性质 | Authority | 公立State / 公立整合State:Integrated / 私立Private |
| A5 | 学生性别 | Gender of Students | 男校/女校/混校，用图标辅助 |
| A6 | 寄宿设施 | Boarding Facilities | 有/无，家长非常关注 |
| A7 | 学校官网 | School Website | 可点击跳转，方便家长深入了解 |

### B. 位置与联系方式（10列）— 学校在哪？怎么联系？

| 序号 | 中文名 | CSV原始列名 | 展示建议 |
|------|--------|-------------|----------|
| B1 | 街道地址 | Street | 与B2、B3合并为一行完整地址 |
| B2 | 区/街区 | Suburb | 合并到地址行 |
| B3 | 城市 | Town / City | 合并到地址行 |
| B4 | 邮政地址 | Postal Address | 与B5、B6、B7合并为一行邮寄地址 |
| B5 | 邮政区 | Postal Address Suburb | 合并到邮寄地址 |
| B6 | 邮政城市 | Postal Address City | 合并到邮寄地址 |
| B7 | 邮编 | Postal Code | 合并到邮寄地址 |
| B8 | 城乡属性 | Urban/Rural | 大城市/小城镇/乡村 |
| B9 | 电话 | Telephone | 可点击拨打 |
| B10 | 邮箱 | Email^ | 去掉"^"符号，可点击发信 |

### C. 学生规模与构成（9列）— 学校大不大？都是什么学生？

这是家长/留学生家庭**核心关注**的数据，尤其是总人数和国际生比例。

| 序号 | 中文名 | CSV原始列名 | 展示建议 |
|------|--------|-------------|----------|
| C1 | 学校总人数 | Total School Roll | 大字加粗，核心指标 |
| C2 | 欧洲裔/白人 | European / Pākehā | 以下6项组成"族裔分布"模块 |
| C3 | 毛利裔 | Māori | 显示人数 + 占比% |
| C4 | 太平洋岛裔 | Pacific | 显示人数 + 占比% |
| C5 | 亚裔 | Asian | 显示人数 + 占比%（留学家庭重点关注） |
| C6 | 中东/拉丁/非洲裔 | MELAA | 显示人数 + 占比% |
| C7 | 其他族裔 | Other | 显示人数 + 占比% |
| C8 | 国际学生 | International | 单独突出，留学家庭核心关注 |
| C9 | 招生计划 | Enrolment Scheme | 有学区限制(Yes) / 无限制(No) |

### D. 学校质量与特色（11列）— 这学校好不好？有什么特色？

帮助家长做最终决策的辅助信息，以及不常用但仍有参考价值的字段。

| 序号 | 中文名 | CSV原始列名 | 展示建议 |
|------|--------|-------------|----------|
| D1 | 公平指数 | Equity Index (EQI) | 数值越低=社区越富裕，解读：<400优越 / 400-500中等 / >500较弱 |
| D2 | 偏远指数 | Isolation Index | 数值越高越偏远，0=市中心 |
| D3 | 教学语言 | Language of Instruction | 英语/毛利语/双语 |
| D4 | 捐赠政策 | Donations | 是否可请求捐赠（反映学校经费来源） |
| D5 | 毛利教育机构 | KME Peak Body | 关联的毛利教育组织（大部分学校为空） |
| D6 | 纬度 | Latitude | 与D7合并为"地理坐标"，用于地图定位 |
| D7 | 经度 | Longitude | 合并展示 |
| D8 | 校长 | Principal* | 展示时去掉"*"符号 |
| D9 | 办学定义 | Definition | 补充说明学校定位 |
| D10 | 运营状态 | Status | 标签样式：🟢运营中 / 🟡筹建中 |
| D11 | 群体入学 | Cohort Entry | 是否支持统一入学时间 |

### E. 行政与区域归属（12列）— 学校归谁管？在哪个学区？

普通家长很少关心，但有行政查询需求时可展开查看。

| 序号 | 中文名 | CSV原始列名 | 展示建议 |
|------|--------|-------------|----------|
| E1 | 教育大区 | Education Region | 如"Tāmaki Herenga Manawa（奥克兰）" |
| E2 | 毛利语区域 | Takiwā | 毛利文化区域划分 |
| E3 | 领土管理局 | Territorial Authority | 如"Albert-Eden Local Board Area" |
| E4 | 区域议会 | Regional Council | 如"Auckland Region" |
| E5 | 地方办事处 | Local Office | 教育部的地方办公室 |
| E6 | 普通选区 | General Electorate | 国会选区 |
| E7 | 毛利选区 | Māori Electorate | 毛利裔专属选区 |
| E8 | 社区编码 | Neighbourhood (SA2 code) | 统计区域代码 |
| E9 | 社区名称 | Neighbourhood (SA2) | 统计区域名称 |
| E10 | 地方选区 | Ward | 地方政府选区 |
| E11 | 学习社区ID | Community of Learning ID | 学校联盟编号 |
| E12 | 学习社区名称 | Community of Learning Name | 学校联盟名称 |

## 四、与 V1 方案的主要区别

| 对比项 | V1（旧方案） | V2（新方案） |
|--------|-------------|-------------|
| 分类数量 | 6类 | **5类**（更简洁） |
| 分类逻辑 | 按数据属性分 | **按家长决策流程分** |
| A类身份概览 | 10列（含校长、状态、定义等） | **7列**，精简为最常用字段，官网提升至A类 |
| 校长/定义/状态/群体入学 | 放在A类（不常查看） | 移至**D类质量与特色**（需要时展开查看） |
| 学校官网 | 放在B类联系方式 | 提升至**A类身份概览**（家长高频使用） |
| 检查报告 | 包含在内 | **移除**（CSV中不存在此列） |
| 展示示例 | 无 | **有真实数据示例** |


## 五、展示交互建议

1. **默认展开**：A（身份）、B（位置联系）、C（学生构成）— 这是家长最需要的信息
2. **点击展开**：D（质量特色）、E（行政归属）— 需要深入了解时再看
3. **族裔分布**用条形图/进度条可视化，比纯数字更直观
4. **国际学生**单独突出显示，这是留学家庭的第一关注点
5. **EQI公平指数**附带简短解读，因为大多数家长不知道这个指标的含义
